### Big feature #2 — Credential-vault migrate arc (M263–M265) — COMPLETE
The first concrete piece of "migrate tooling": detect-then-upgrade for the
on-disk credential vault.
- **M263 (kernel)** — `creds.InspectVault` reads the envelope's KDF/iteration
  params *without decrypting*; `Store.MigrateEncryption` re-encrypts a stale
  vault (legacy keyed-HMAC KDF or low iteration count) at the current PBKDF2
  policy, secrets + passphrase unchanged.
- **M264 (CLI)** — `agt vault migrate` wires it: no-op (clear notice) for
  plaintext / already-current; requires the passphrase only for a real
  re-encryption; prints before→after KDF + iterations.
- **M265 (legibility)** — `agt vault status` now surfaces the KDF policy +
  up-to-date flag (passphrase-free), with a "migration: recommended" pointer
  when stale.
- **Loop closed**: see-it's-stale (M265) → fix-it (M264) → shared passphrase-free
  detection (M263).

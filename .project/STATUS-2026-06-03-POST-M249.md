## Vision arc (M241–M249) — the big one this session
A documented feature (`agt run --image`, M91/M93) that was **completely
non-functional** end to end is now working in **both directions** across every
surface:

| Direction | Milestones | Surface |
|---|---|---|
| Send (provider emission) | M241–M245 | Anthropic, OpenAI, Gemini, Bedrock/Claude, Vertex (both dialects); compat vendors inherit |
| API input | M246 | `/v1/chat/completions` accepts `image_url` parts |
| Inbound channels | M247–M249 | Telegram photo, Slack image file, Discord slash ATTACHMENT |

- Provider-agnostic `data:` URL carrier (no wire/type change — `images` stays
  `[]string`); allowlist-gated inbound fetch (no file-deref for unauthorized
  senders); 12 MiB cap vs the 16 MiB control-plane request limit.
- **+35 tests** across the 9 milestones (1782 → 1817).

## What's done vs. what remains

directions; channel delivery (chunking + empty-guard, all 3); provider/catalog
vendor coverage; redaction; OpenAI-API streaming + usage; `agt run` exit codes
(verified correct — no bug); unreachable-daemon CLI error.

**Known remaining nits** (small, evidence-based follow-ups):
1. Responses-API `input_image` input (chat completions done; Responses is a
   different part shape) — completes vision input symmetry.
2. API/channel run paths don't pre-gate vision capability (a non-vision model +
   image surfaces a provider error rather than a clean pre-check).
3. Discord inbound journals the message before the (post-ACK) image fetch, so
   the inbound event doesn't record the image count.

**Bigger, not-yet-started** (the original "vision" preview list): SDK,
marketplace, migrate tooling, ambient/microVM. These are features, not bugs;
deferred in favour of the user-felt-bug rhythm.

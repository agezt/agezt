# AGEZT — Security Architecture & Reconnaissance Map

> Generated by `sc-recon` (Phase 1). Repo root: `D:\Codebox\PROJECTS\AGEZT`.
> AGEZT is a self-hostable multi-agent platform: a Go kernel/daemon (`agezt`) + thin Go CLI client (`agt`),
> a React/TS web console embedded into the daemon, Go/TS/Python/Rust SDKs, and ~70 in-process plugins
> (providers, ~34 comm channels, tools incl. code execution). It is a **high-sensitivity application**:
> it executes arbitrary code, makes outbound calls on behalf of agents, ingests untrusted messages from
> ~34 channels, speaks MCP (stdio + remote HTTP), and stores provider/OAuth secrets in an encrypted vault.

- **Frontend (`frontend/package.json`):** React 19, Vite 8, TypeScript 6, Tailwind 4, Radix UI primitives, `@xyflow/react` (React Flow — Flow Studio), `lucide-react`. Test: Vitest + Playwright (e2e). ### Build Tools / Package Managers
- Go modules (`go.mod` / `go.sum`, Go 1.26.4), `Makefile`.
- npm/pnpm for frontend (`package-lock.json` + `pnpm-lock.yaml` both present), Vite build.
- pyproject (Python SDK), Cargo (Rust SDK).

### Databases / Persistence / ORMs
- **No SQL/NoSQL database and NO ORM.** No `database/sql`, sqlite, bbolt, postgres, mongo, redis drivers found.
- Persistence is **append-only JSONL journal + JSON state buckets on the local filesystem** under the AGEZT home dir (`~/.agezt` by default; isolated `AGEZT_HOME` for tests):
  - `kernel/journal/journal.go` (`json.Marshal` per event, line 207/363) — event log.
  - `kernel/state/state.go` (`json.MarshalIndent`, line 207) — keyed state buckets, 0600 perms.
  - `kernel/datalake/datalake.go` — derived/queryable views.
  - Encrypted secrets vault: `kernel/creds/creds.go` → `<BaseDir>/creds.json` (0600).
- Implication: **no SQL-injection surface**; data-store risk is path traversal / file permissions / JSON deserialization, not query injection.

## 2. Application Type Classification

**Hybrid: long-running daemon + CLI + embedded web app + library SDKs.**

| Dimension | Classification |
|-----------|----------------|
| Primary | **Self-hosted autonomous-agent daemon** (`cmd/agezt`) hosting an agent runtime + plugin fleet |
| Control surface | **CLI tool** (`cmd/agt`, 63 subcommands) over a localhost token RPC control plane |
| Web surface | **Web Application + REST API** — React SPA console (`kernel/webui`) + native REST (`kernel/restapi`) + **OpenAI-compatible API** (`kernel/openaiapi`) |
| Integration surface | **Message/channel consumers** (~34 channels), **MCP client** (stdio + remote HTTP), **agent gateway** (`kernel/agentgw`, Unix socket/TCP) |

Not serverless, not containerized (no Dockerfile/k8s/terraform — see §8). Designed to run as a local/loopback daemon, optionally fronted by a reverse proxy/tunnel.

### 3b. HTTP routes

**Web UI (`kernel/webui/webui.go:611-669`)** — every data route wrapped in `s.auth`; the table is data-driven:
- Token-free (deliberately): `/` SPA shell (`shellAuth`), `/api/authmeta`, `/api/login`, `/api/logout`, `/assets/`, `/favicon.ico` — all still pass `s.secure` (host + same-origin + CSP).
- Token-gated: `/events` (SSE, accepts `?token=` because EventSource can't set headers), **272 control-plane commands** proxied via `apiRoutes` / `readArgsRoutes` / `writeRoutes` / `jsonRoutes` tables (e.g. `/api/skill/import`, `/api/world/edit`, `/api/redact/test` `:587-592`).

- **OAuth redirect (token-free):** `/oauth/callback` → `handleOAuthCallback` (`:668/675`) — security rests on the minted `state`.

**REST API (`kernel/restapi/restapi.go:173-197`):**
- Unauth: `/healthz`, `/readyz`.
- Token (`s.auth`): `/metrics`, `/api/v1/health`, `/api/v1/models`, `/api/v1/runs`, `/api/v1/runs/`.
- **Admin token (`s.adminAuth`):** `/api/v1/mailbox/*` (messages/inbox/watch/topics), `/api/v1/update` + `/api/v1/update/apply`.

**OpenAI-compatible API (`kernel/openaiapi/openaiapi.go:169-175`):** `POST /v1/chat/completions`, `POST /v1/responses`, `GET /v1/models`, `GET /v1/models/{id}`, `POST /v1/audio/transcriptions` — all `s.auth` (bearer). Lets any OpenAI SDK/IDE drive AGEZT — **untrusted prompt ingress that reaches the agent runtime & tools.**

**Agent gateway (`kernel/agentgw/gateway.go:119-151`):** `/v1/eventbus/{subscribe,publish}`, `/v1/memory/{write,delete,search}`, `/v1/log/{read,write}`, `/v1/agent/{list,query}`, `/v1/token/create`, `/v1/config/*`, `/health` (no auth). All but `/health` wrapped in `withAuth` (`:216`); 1 MiB body cap; per-token sliding-window rate limit (≤4096 entries, CWE-770 bound).

### 3c. CLI commands
- **`cmd/agezt`** — daemon binary: bare run / `daemon` / `version` / `help`. Reads many `AGEZT_*` env vars; provider selection requires real keys (no default provider/model — see memory).
- **`cmd/agt`** — 63 registered subcommands (`cmd/agt/cmd_register.go`): `run`, `halt`, `resume`, `agent`, `approvals`, `redact`, `config`, `channel`, `acp`, `budget`, `catalog`, `backup`, `memory`, `standing`, `journal`, `why`, `doctor`, `market`, `configcenter`, etc. All are thin clients over the control-plane RPC (read addr+token from runtime files).

### 3d. Message / channel consumers (untrusted external ingress — see §4)

- **Inbound patterns:** long-poll (telegram, matrix), webhook HTTP endpoints (slack, discord, whatsapp, line, feishu, wecom, etc.), socket (irc, twitch, nostr). Each `Start()` runs on a daemon goroutine; inbound dispatched under `channel.Guard` (panic recovery) and an **allowlist (fail-closed)** before becoming an agent run.

### 3e. Scheduled tasks
- `kernel/cadence/cadence.go` (interval/once/continuous schedules, DST-safe; `injection.go` target validation) and `kernel/scheduler/scheduler.go` + `nodes.go` — drive standing orders / system tasks / workflows. ### 3f. WebSocket / SSE
- **SSE:** `/events` (web UI), mailbox watch (`/api/v1/mailbox/watch`), event-bus subscribe (agentgw `/v1/eventbus/subscribe`).
- **WebSocket:** `coder/websocket` dep present (channel/tunnel transports).

## 5. Trust Boundaries

### Authentication / authorization
- **Control plane:** primary 32-byte hex token, **constant-time** compare (`tokenIsPrimary`, `server.go:325`); tenant tokens via `tenant.Registry.Authorize` (constant-time), restricted to tenant-safe command set (`server.go:485-492`).
- **Web UI:** `auth()` = `secure()` + `authorized()` (`webui.go:1040`). `authorized` (`:1201`): token-only when no password; password mode = token OR session (or token AND session in `AGEZT_WEB_PASSWORD_STRICT`). Tokens constant-time (`tokenMatch`, `:1217`).
- **REST:** `s.auth` (bearer) + `s.adminAuth` for mailbox/update.
- **OpenAI API / agentgw:** bearer token / HMAC-SHA256 JWT (HS256 + iss/aud pinned, `agentgw/token.go`).

### Same-origin / CSRF / host controls (`webui.go:1022-1134`)
- `secure()` applies on **every** route: `setSecurityHeaders` → `hostAllowed(r.Host)` (allowlist via `AGEZT_WEB_ALLOWED_HOSTS`; loopback/IP literals allowed) → `sameOriginMutation(r)`.
- **CSRF defense:** `sameOriginMutation` (`:1117`) rejects state-changing methods with cross-site `Sec-Fetch-Site` or a mismatched `Origin` host; GET/HEAD/OPTIONS exempt. Session cookie `SameSite=Strict, HttpOnly, Secure` (proxy-aware via `X-Forwarded-Proto`, `session.go:211-217`).
- **No CORS allow-all:** there is **no permissive `Access-Control-Allow-Origin`**; the model is same-origin-only. CSP is locked down (see §9).

### Rate limiting / lockout
- Web UI password login: failed-attempt **lockout** (`s.sessions.lockedOut()` → 429, `session.go:188`), body capped (`loginBodyLimit`).
- Agent gateway: per-token sliding-window rate limit, bounded entry table (CWE-770), idle eviction.
- `kernel/streamlimit` caps streaming responses; slow-loris timeouts via `newGuardedHTTPServer` (`main.go:4288`).

### Input validation
- MCP server config: name regex `^[a-z][a-z0-9]{0,15}$`, env-key/header-name regexes (`kernel/mcp/store.go:99-108`) — blocks injection via env keys/headers.
- Channel account labels validated (`settings.ValidAccountLabel`).
- Body size caps everywhere (web UI 1 MiB, agentgw 1 MiB, oversize-request tests in restapi/openaiapi).
- **Prompt-injection guard** (`kernel/agent/observation.go`) — the agent-specific trust boundary; tri-state `AGEZT_PROMPT_INJECTION_GUARD` (`kernel/runtime/runtime.go:2162`), "Trust web content" per-session toggle in chat UI.

## 7. Authentication Architecture

| Surface | Pattern | Storage / details |
|---------|---------|-------------------|
| Control plane | **API token** (32-byte hex, per-boot random) | written to runtime files (addr+token) read by `agt`; constant-time compare; tenant tokens via registry |
| Web console | **API token + optional password session** | token from daemon banner URL (`?token=` for shell/EventSource only); password = `AGEZT_WEB_PASSWORD`/Config Center; session cookie `HttpOnly`+`SameSite=Strict`+`Secure(proxy-aware)`; constant-time password compare + failed-attempt lockout (`session.go`). Strict mode = token AND session. |
| REST / OpenAI API | **Bearer token** (`s.auth`) + admin tier (`adminAuth`) | same daemon token |
| Agent gateway | **HMAC-SHA256 JWT** | HS256 pinned (alg-confusion closed), iss/aud `agezt-agentgw` pinned, expiry, per-token caps + rate, mint-from-parent capped (`/v1/token/create`) |
| Provider OAuth | **OAuth 2.0 / PKCE** | ChatGPT/Codex (`kernel/chatgptauth`, PKCE+state, refresh-on-skew), channel OAuth (Slack/Mastodon, opaque 32-byte state, netguard-guarded token exchange) → tokens in **encrypted vault** |
| Vault (secrets at rest) | **AES-256-GCM**, **PBKDF2-HMAC-SHA256 (200k iters)**, 32-byte salt + 12-byte nonce per save | `kernel/creds/encrypt.go`; passphrase precedence: `AGEZT_VAULT_PASSPHRASE` → **machine-derived** (machine GUID/`/etc/machine-id`/IOPlatformUUID + uid + username, SHA-256) → plaintext fallback; boot-time auto-encrypt migration (`machine.go`, opt-out `AGEZT_VAULT_AUTOENCRYPT=off`). Multi-key keyring per env var (`#label`, active-key flag). |

No password user-database / MFA / account model beyond the single operator console password. Authentication is operator-to-daemon, not multi-user.

## 9. Detected Security Controls

| Control | Implementation |
|---------|----------------|
| **SSRF guard** | `kernel/netguard/netguard.go` — **dialer-level** IP validation on initial connect + every redirect hop; blocks loopback/RFC1918/link-local(169.254 metadata)/CGNAT/multicast + IPv6-smuggled v4; `OnBlock` → `netguard.blocked` journal event. Used by http/fetch/web_search tools, MCP HTTP, OAuth exchanges. |
| **Sandbox / process isolation** | `kernel/warden` — profiles None/Namespace/Container/MicroVM; timeout + `WaitDelay`, output cap (256 KiB), workdir scoping, allowlist-only env, Linux `Setpgid`+`Prlimit` (CPU 120s, AS 2 GiB, 512 FDs, 256 MiB file). `code_exec` adds ephemeral scratch dir + Deno FS jail. Honest isolation reporting in events. |
| **Secret redaction** | `kernel/redact/redact.go` — literal scrub (provider keys, ≥8 bytes, longest-first) + pattern scrub (sk-/AKIA/ghp_/xox/AIza/JWT/PEM/bearer/connection-string pw) → `[REDACTED]`; deterministic. Applied to tool output & logs. |
| **Env scrubbing** | `kernel/envscrub` + `kernel/mcp/client.go:316` — child processes/MCP get allowlist-only env; drops anything containing KEY/TOKEN/SECRET/PASSWORD/CRED/AWS_/AGEZT_. |
| **Prompt-injection guard** | `kernel/agent/observation.go` — taints untrusted tool output (browser/http/fetch/web_search/mcp_*/forge_*), wraps in UNTRUSTED OBSERVATION, detects directive-like phrases, 1-turn taint window forces approval on next effectful action. Tri-state `AGEZT_PROMPT_INJECTION_GUARD`; per-session "Trust web content" toggle. |
| **Capability / trust ladder** | `kernel/edict/edict.go` — non-overridable hard-deny rules (fork-bomb, `rm -rf /`, mkfs, shutdown, audit-disable) + L0..L4 ladder + AskPolicy (allow/deny/prompt). Default posture: many caps LevelAllow (default-allow design law); shell/file-write/http default Ask. |
| **HITL approval gate** | `kernel/approval/approval.go` — suspends tool loop, emits `approval.requested` with effect class/rollback/regret metadata, blocks on operator decision (`agt approve/deny`) with 5-min auto-deny timeout; full journal trail. |
| **Audit logging** | `kernel/agentgw/audit.go` (every authorized gateway request journaled pre-handler) + journal events for warden exec, netguard blocks, approvals, channel inbound — `agt why` walks correlation chains. |
| **Security headers / CSP** | `setSecurityHeaders` (`webui.go:1085`): `default-src 'none'`, `script-src 'self'`, `connect-src 'self'`, `frame-ancestors 'none'`, `X-Frame-Options DENY`, `Referrer-Policy no-referrer`, `nosniff`, `base-uri/form-action 'none'`. |
| **Host / origin allowlist + CSRF** | `hostAllowed` + `sameOriginMutation` (§5). |
| **Constant-time auth + lockout** | `subtle.ConstantTimeCompare` for all tokens/passwords; login lockout; slow-loris timeouts. |
| **Vault encryption at rest** | AES-256-GCM + PBKDF2-200k + machine-binding (§7). |
| **Signed update / channel verification** | Ed25519/secp256k1 (btcec) for update + channel signatures (Discord Ed25519, webhook signing). |

Notable historical context (from project memory, not re-verified here): `agentgw` was hardened after a hardcoded-token + unauth token-mint finding; Ed25519 update signature verification shipped but may require `DefaultPublicKeyHex` activation (release-eng step). Worth confirming downstream.
